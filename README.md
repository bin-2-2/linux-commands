# 리눅스 프로세스 관리: 상황별 명령어 정리

## 1. 개요
리눅스 서버를 다루다 보면 프로그램이 멈추거나 시스템이 느려질 때가 있습니다. 이때 **진단(top, ps)**하고 **처치(jobs, kill)**하는 필수 명령어 4가지를 정리했습니다.

> **핵심 목표:** 프로세스의 상태를 파악하고 불필요한 작업을 종료할 줄 안다.

## 2. 실시간 모니터링: top
시스템이 갑자기 느려졌을 때 **"범인"을 실시간으로 잡는 도구**입니다.
* **특징:** CPU와 메모리 점유율이 높은 순서대로 프로세스가 갱신됩니다.
* **주요 단축키 (실행 중):**
    * `q`: top 종료
    * `k`: 프로세스 종료 모드 (PID 입력)
    * `1`: 모든 CPU 코어별 사용량 보기

## 3. 특정 순간 포착: ps (Process Status)
현재 실행 중인 프로세스의 **스냅샷(사진)**을 찍어서 확인합니다. `grep`과 함께 사용하여 특정 프로그램을 찾을 때 유용합니다.

## 4. 백그라운드 작업 관리: jobs
터미널 하나에서 여러 작업을 동시에 돌릴 때, **"뒤에서(Background)"** 실행 중이거나 멈춘 작업들을 확인합니다.

* **사용 목적:** `Ctrl+Z`로 작업을 멈췄거나 `&`를 붙여 백그라운드로 보낸 작업의 리스트를 볼 때 사용합니다.
* **출력 정보 구조:** `[작업번호] 상태 명령어`
* **기호의 의미:**
    * `+`: 현재 처리 대상 (Current) - `fg` 명령 입력 시 우선 대상
    * `-`: 바로 이전 작업 (Previous)

## 5. 프로세스 종료: kill
응답 없는 프로그램을 **강제로 종료**하거나 특정 신호(Signal)를 보낼 때 사용합니다.

* **주의사항:** 무조건 끄는 게 능사가 아닙니다. 상황에 맞는 옵션(Signal)을 써야 데이터 손실을 막습니다.
* **옵션 비교 (실무 팁):**

| 상황 | 명령어 예시 | 설명 |
|:---:|:---|:---|
| **일반 종료** | `kill -15 [PID]` | (SIGTERM) 프로그램에게 "종료해 줄래?"라고 정중하게 요청합니다. (저장할 시간을 줌) |
| **강제 종료** | `kill -9 [PID]` | (SIGKILL) **즉시 사살.** 저장하지 않고 메모리에서 바로 날려버립니다. |
| **작업 번호로** | `kill %1` | PID를 모를 때 jobs에서 확인한 **작업 번호(1)**로 종료할 수 있습니다. |

**[자주 쓰는 옵션 조합]**
```bash
# 모든 프로세스를 상세하게 출력 (BSD 스타일)
$ ps aux

# 특정 프로세스(예: python)만 찾고 싶을 때
$ ps -ef | grep python
